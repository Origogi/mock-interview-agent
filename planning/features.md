# Tech-Interviewer AI 기능 명세

## 1. 제품 개요

Tech-Interviewer AI는 사용자의 이력서를 바탕으로 실제 기술 면접처럼 개인화된 질문과 꼬리 질문을 제공하고, 면접 종료 후 답변별 피드백과 종합 리포트를 제공하는 AI 면접 트레이닝 서비스다.

기존 면접 대비 챗봇은 정형 질문을 반복하는 경우가 많다. 이 서비스는 사용자의 실제 이력서에 드러난 프로젝트, 기술 스택, 설명이 부족한 지점을 바탕으로 질문을 만들고, 답변 수준에 따라 다음 질문의 방향을 바꾼다.

### 핵심 가치

1. **개인화:** 모든 면접은 사용자의 이력서와 이전 답변을 기준으로 진행된다.
2. **긴장감:** 실제 면접처럼 답변의 빈틈을 파고드는 꼬리 질문을 제공한다.
3. **직관적 피드백:** 사용자는 점수, 문항별 피드백, 종합 리포트, 다시 답변하기 흐름으로 바로 학습할 수 있다.

## 2. 사용자 흐름

### Page 1. 이력서 업로드

사용자는 PDF 이력서를 업로드해 면접을 시작한다. 서비스는 이력서 내용을 분석해 이후 질문의 기준이 되는 면접 컨텍스트를 만든다.

사용자에게 제공되는 기능:

- 이력서 PDF 업로드
- 드래그 앤 드롭 업로드
- 업로드 및 분석 진행 상태 표시
- 서비스 상태 안내
- 샘플 이력서로 빠르게 시작
- 다크 톤의 집중형 첫 화면

### Page 2. 이력서 요약

사용자는 면접 시작 전에 서비스가 이력서를 어떻게 이해했는지 확인한다. 이 화면은 면접관이 어떤 관점으로 질문할지 미리 보여주는 역할을 한다.

사용자에게 제공되는 정보:

- 주요 기술 스택
- 핵심 프로젝트 경험
- 강점으로 보이는 지점
- 추가 검증이 필요한 공격 포인트
- 경력, 기술, 프로젝트, 공격 포인트 요약 수치

이 화면에서 사용자는 분석 결과를 확인한 뒤 실전 면접으로 진입한다.

### Page 3. 실전 면접

실전 면접 화면은 이 서비스의 핵심 경험이다. 사용자는 면접관의 질문에 답변하고, 서비스는 답변을 평가한 뒤 다음 질문을 이어간다.

사용자에게 제공되는 기능:

- 이력서 기반 첫 질문
- 총 20문항, 4개 세션 고정 순서의 면접 진행
- 세션 안에서는 이전 답변과 평가를 반영한 동적 꼬리 질문
- 면접관 질문이 실시간으로 작성되는 듯한 표시
- 긴 답변 작성에 적합한 멀티라인 입력
- 답변 제출 후 현재 답변과 면접관 반응을 함께 보기 쉬운 스크롤 경험
- 답변 평가 완료 알림
- 4세션 사이드바의 면접 진행 상태
- 문항별 점수와 피드백 카드
- Best / Good / Bad 3단계 점수 톤
- 면접 조기 종료
- 이전 질문으로 되감아 다시 답변하기
- 샘플 답변 채우기

### Page 4. 최종 리포트

최종 리포트는 사용자가 면접을 복기하고 다음 연습 방향을 결정하는 화면이다.

사용자에게 제공되는 기능:

- 종합 점수
- 역량별 점수 시각화
- 세션별 요약 카드
- 강점, 약점, 개선 방향
- 세션별 문항 질문, 답변, 피드백 상세
- 미완료 세션의 평가 부족 표시
- 조기 종료 리포트 표시
- 특정 질문 다시 답변하기
- 면접 다시 시작

## 3. 구현된 핵심 기능

### F1. 이력서 기반 맞춤 질문

서비스는 사용자의 이력서를 분석해 면접 질문의 기준을 만든다. 사용자가 실제로 경험한 프로젝트와 사용 기술을 중심으로 질문하기 때문에, 정형 면접 질문보다 더 개인화된 연습이 가능하다.

이 기능이 제공하는 가치:

- 사용자는 본인의 실제 이력서로 면접을 연습할 수 있다.
- 면접 질문이 사용자의 프로젝트와 기술 스택에 맞춰진다.
- 이력서에서 설명이 부족하거나 검증이 필요한 지점을 질문 소재로 삼는다.
- 면접 전에 분석 요약을 확인할 수 있어 서비스가 어떤 맥락으로 질문할지 이해할 수 있다.

### F2. 20문항/4세션 면접 구조

면접은 총 20문항이며, 4개 세션을 고정 순서로 지난다. 세션 순서는 바뀌지 않고, 각 세션 안의 질문 내용만 사용자의 이력서, 이전 답변, 평가를 반영해 동적으로 생성된다.

세션 구성:

- `cs_fundamentals` / CS Fundamentals / Q1~Q5
- `framework_usage` / Framework Usage / Q6~Q10
- `problem_solving` / Problem Solving / Q11~Q15
- `communication` / Communication / Q16~Q20

사용자 경험:

- 첫 질문은 이력서 기반으로 시작한다.
- 이후 질문은 현재 세션의 범위 안에서 이전 답변과 평가를 반영한다.
- 답변이 모호하면 구체적인 사례나 근거를 요구하는 질문이 이어진다.
- 답변이 충분하면 다른 검증 포인트로 자연스럽게 넘어간다.
- 한 세션의 5개 문항을 완료하면 다음 세션으로 넘어간다.
- 한 번에 하나의 질문만 제공해 실제 면접 흐름을 유지한다.

### F3. 답변별 즉시 평가

사용자는 각 답변을 제출할 때마다 점수와 피드백을 받는다. 이 피드백은 최종 리포트까지 기다리지 않아도 현재 답변의 수준을 바로 파악하게 해준다.

평가에서 확인하는 요소:

- 질문에 직접 답했는지
- 기술적으로 정확한지
- 실무 경험이 구체적인지
- 문제 해결 과정이 논리적인지
- 개선 방향이 명확한지

점수 톤:

- **Best:** 충분히 구체적이고 실무 판단이 드러나는 답변
- **Good:** 방향은 맞지만 구체성이나 깊이가 부족한 답변
- **Bad:** 모호하거나 질문의 핵심을 해결하지 못한 답변

### F4. 실시간 면접관 응답

면접관의 질문과 반응은 한 번에 나타나지 않고, 실제로 생각하며 말하는 것처럼 순차적으로 표시된다. 이 경험은 사용자가 일반 챗봇보다 더 실제 면접에 가까운 긴장감을 느끼도록 돕는다.

사용자 경험:

- 면접관의 질문이 자연스럽게 작성되는 듯이 보인다.
- 응답 중에는 사용자가 다음 답변을 서두르지 않도록 입력 흐름이 제어된다.
- 질문이 끝나면 사용자는 바로 답변을 작성할 수 있다.
- 긴 답변을 제출해도 방금 작성한 답변과 면접관의 반응을 이어서 볼 수 있다.

### F5. 4세션 진행 사이드바

Page 3의 사이드바는 사용자가 총 20문항 중 현재 어디까지 진행했는지, 각 세션이 어떤 상태인지, 이전 답변들이 어떤 평가를 받았는지 빠르게 확인하게 해준다.

사용자에게 보여주는 정보:

- 현재 세션과 질문 번호
- 각 세션의 문항 범위
- 세션별 완료, 진행 중, 대기 상태
- 전체 진행률
- 완료된 답변 목록
- 완료 세션의 평균 점수
- 문항별 점수
- 문항별 간단 피드백
- 다시 답변할 수 있는 질문 진입점

### F6. 최종 리포트

최종 리포트는 면접 전체를 회고하는 결과 화면이다. 사용자는 단순 평균 점수가 아니라, 어떤 역량이 강하고 어떤 부분을 보완해야 하는지 확인한다.

리포트 구성:

- 종합 점수
- 역량별 점수
- 세션별 요약 카드
- 완료 세션의 평균 점수
- 미완료 세션의 평가 부족 상태
- 강점 요약
- 약점 요약
- 개선 방향
- 세션별 상세 문항 피드백
- 질문별 다시 답변하기

세션 점수 정책:

- 세션의 5개 답변이 모두 완료되면 해당 답변 점수의 단순 평균으로 세션 점수를 표시한다.
- 세션의 5개 답변이 완료되지 않으면 점수를 확정하지 않고 평가 부족으로 표시한다.

리포트의 목표:

- 사용자가 다음 연습에서 무엇을 고쳐야 하는지 알게 한다.
- 단순한 채점이 아니라 면접 준비 행동으로 이어지게 한다.
- 답변 하나하나를 다시 열어보고 복기할 수 있게 한다.

### F7. 면접 조기 종료

사용자가 20문항 전체를 완료하지 못해도, 가능한 범위에서 학습 가치를 회수할 수 있도록 조기 종료 기능을 제공한다.

정책:

- 유효 답변이 5개 이상이면 부분 리포트를 제공할 수 있다.
- 유효 답변이 5개 미만이면 평가가 부족하다고 안내하고 리포트를 만들지 않는다.
- 완료된 세션은 세션 요약과 상세 피드백을 제공한다.
- 미완료 세션은 평가 부족으로 표시한다.
- 실수로 종료하지 않도록 확인 모달을 제공한다.
- 부분 리포트에는 신뢰도 제한을 명확히 안내한다.

사용자 가치:

- 시간이 부족해도 지금까지의 답변을 버리지 않는다.
- 짧은 연습에서도 의미 있는 피드백을 얻을 수 있다.
- 리포트가 충분하지 않은 경우에는 잘못된 확신을 주지 않는다.

### F8. 타임머신: 이전 질문 다시 답변하기

타임머신은 사용자가 특정 질문으로 되돌아가 다시 답변할 수 있는 기능이다. 실제 면접 후 자주 생기는 "이 질문에는 이렇게 답했어야 했는데"라는 후회를 학습 행동으로 연결한다.

사용자 흐름:

1. 완료된 질문 카드 또는 리포트의 문항 상세에서 다시 답변하기를 선택한다.
2. 선택한 질문 이후의 답변과 평가는 다시 생성된다는 안내를 확인한다.
3. 화면 전체에 타임머신 진행 상태가 표시된다.
4. 선택한 질문이 현재 질문으로 다시 열린다.
5. 사용자가 새 답변을 제출하면 이후 면접 흐름과 리포트가 새 답변 기준으로 이어진다.

정책:

- 선택한 질문 이전의 답변과 평가는 유지한다.
- 선택한 질문부터 Q20까지의 답변, 평가, 리포트는 무효화하고 새로 만든다.
- 세션 경계와 관계없이 선택한 질문 이후의 면접 흐름은 새 답변 기준으로 다시 이어진다.
- 사용자는 같은 질문에 더 나은 답변을 시도할 수 있다.
- 현재 면접 흐름을 크게 방해하지 않도록 전용 오버레이로 전환 상태를 보여준다.

### F9. 샘플 이력서 빠른 진입

개발 검증과 데모를 빠르게 하기 위해 샘플 이력서로 바로 시작하는 기능을 제공한다.

사용자 경험:

- 이력서 파일 없이도 요약 화면으로 진입할 수 있다.
- 전체 면접 흐름을 빠르게 확인할 수 있다.
- 실제 사용자 플로우와 같은 화면 구조로 이어진다.

### F10. 샘플 답변 채우기

면접 중 답변 입력을 빠르게 테스트하기 위한 기능이다. 사용자는 Best, Good, Bad 수준의 샘플 답변을 생성해 평가 결과와 리포트 반영을 확인할 수 있다.

정책:

- 샘플 답변은 자동 제출되지 않는다.
- 사용자가 내용을 확인한 뒤 직접 제출한다.
- 답변 수준별 평가 톤이 일관되게 보이는지 확인하는 데 사용한다.
- 버튼, 드롭업, 로딩 표시가 포함되어 있어 면접 화면 안에서 가볍게 사용할 수 있다.

### F11. 일관된 시각 피드백

전체 화면은 어두운 배경, 유리 질감, 부드러운 전환, 점수별 색상 톤을 사용해 면접의 집중감과 결과 화면의 완성도를 높인다.

사용자 경험:

- 페이지 전환이 부드럽다.
- 결과 차트와 점수 표시가 단계적으로 등장한다.
- 아코디언이 자연스럽게 펼쳐진다.
- 점수 단계가 토스트, 카드, 리포트에서 일관되게 보인다.
